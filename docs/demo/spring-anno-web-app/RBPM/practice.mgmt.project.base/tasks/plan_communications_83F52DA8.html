Task: Plan Communications

Purpose

Determine the information and communications needs of the project stakeholders (who needs what, when and how).

Procedure

The formality of communications planning varies depending on the size, length, and complexity of the project, but the key lies in the thinking process to determine the information and communication needs of the projectstakeholders (who needs what, when and how). Typically, it involves the following:

  1. Identify the stakeholders and their interests in the projector programme, segmenting them into logical groups.

    One useful way is to create a stakeholder matrix which maps each of the stakeholders against his/her particular interest in the project or programme. The matrix should also indicate whether a stakeholder is active in the projector programme, interested in it, an authorisation party, and so on.

    Where necessary, further analysis on the stakeholders can be done (for example, attitude to change, sphere of influence, and so on), but this is only done on a need basis where it will help to make the communication to and from these stakeholders more effective. See Identify Stakeholders for more information.

  2. Based on the analysis above, determine what needs to be communicated to and from the stakeholders in terms of:
    • the type of information (for example, schedule, status, progress, issues etc).
    • the means of the communication (for example, meetings, presentations, progress reports, newsletters, intranet, collaboration tools, and so on; see also 'Project Meetings' and 'Project Progress Reporting' which are common forms of project communications).
    • frequency
    • responsible parties.
  3. Document the results of the thinking above (see Guidelines below).

Guidelines

There is no fixed format for the Communications Management Plan and it can range from a few bullet points and/or tables to a comprehensive document. Depending on the size and complexity of the project or programme, the communications plan should detail:

  • Message to be communicated, for example, progress, early warning of emerging issues.
  • Stakeholder or stakeholder group
  • Date for communication
  • Method of communication, for example, informal discussion, formal presentation.
  • Who is responsible for ensuring the communication takes place.
  • What information/feedback the stakeholder can give back to the programme or project.

Where the communications strategy also includes the enabling of communications channels, then the communications plan needs to cover the planned implementation of the identified channels. For example, a plan would be required for implementing instant messaging for all members of a geographically dispersed project team.
