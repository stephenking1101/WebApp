Task: Manage Project Execution

Steps

Manage Scope and Requirements
Changes to scope and requirements are a common threat to projects. Project managers must watch for “scope creep.”. An increase in scope that doesn’t include a corresponding adjustment to project cost, effort or timeline may result in the project being delivered late or over budget.

See Change Control.
Manage Schedule

Managing the project schedule helps the project stay on track to deliver what was agreed.

Procedure

The project schedule must be regularly updated with actual effort expended and latest estimated effort to complete the tasks.

On a weekly basis:

  1. Get a progress update from all project participants.
  2. Update the status, effort expended and the current estimate to complete on all current tasks.
  3. Update the status of milestones due to be completed.
  4. Adjust any dependencies altered since the last update.
  5. Reschedule taking account of these updates.
  6. Assess the impact of progress made to the project delivery schedule and take mitigating action as appropriate
Manage Costs

An essential part of a project manager’s job is to ensure that the project is completed within the approved budget.

Procedure

  1. Identify budget variances
  2. Understand the reasons for positive or negative variances in the budget
  3. Make adjustments to the budget to reflect any changes or revise the cost estimates
  4. Assess impacts to project scope and requirements
  5. Get approval from the project sponsor for any budget adjustments
