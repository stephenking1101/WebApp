Task: Documentation Management Approach

Purpose

Planning documentation management for the project. This is an important aspect of project management because project health is often reflected through it's documents. Planning for documentation and artefact management aids collaboration for project teams, governance forums and stakeholders. This also helps to meet compliance and audit requirements effectively.

Prodecure

Project lifecycle involves documentation of information in a variety of templates and tools (e.g. SharePoint, Clarity etc.). It is essential to understand the key considerations for documentation:

  1. The Project Manager is accountable to agree with the sponsor, project team and stakeholders on a shared location or repository (e.g. SharePoint) where all project documents can be stored on an ongoing basis.
  2. Agree on a folder structure which suits the needs of the project and helps to easily locate the information. A recommended structure is to name folders by phases and types, i.e., Initiation, Definition, Execution, Closure, Reports, Communication etc.
  3. Set the right level of access to ensure information availability and protection.
  4. Project manager to ensure compliance with Group Guidance Considerations related to project documentation, e.g. Document Classifications, Records Management, Information Security etc.
  5. Ensure project documents are updated and available at SharePoint throughout the project lifecycle.
  6. Plan audit trails and agree on how the information will be updated in the project documents, i.e. version control, changes, sign offs etc.
  7. Agree on timeliness of documents and availability for audit.

Key Considerations

Consider the requirements for minimum standards and mandatory documents to understand project documentation requirements (both Clarity and Template, e.g. Project Plan, Change Control and Risk & Issues are to be managed in Clarity)
