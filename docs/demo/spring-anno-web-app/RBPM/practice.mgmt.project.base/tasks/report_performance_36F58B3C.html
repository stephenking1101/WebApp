Task: Report Performance

Procedure

Performance reporting involves collecting and disseminating performance-related information to allow comparison of the project or programme to its own historical performance and the performance of other projects or programmes. The information used includes:

Status reporting

describing where the project or programme now stands.

Progress reporting

describing what the projector programme team has accomplished.

Forecasting

predicting future project or programme status and progress



Performance Reporting should provide information on scope, schedule, cost, quality and risk. It also shows relative performance of one project team or development area against others / against own historical trends, as part of ongoing performance improvement.

Inputs to Performance Reporting:

The following are normal inputs to performance reporting

Project Schedule

which contains the baseline for assessing project performance.

Work Results

which should be reported within the framework of the Communications Management Plan

Other Project or Programme Records

which often contain information related to the project or programme context and should hence be considered.


Tools and techniques

Performance Reporting often involves the following tools and techniques:

Performance Reviews

meetings to assess project status or progress (see How to hold a project progress review meeting)

Variance Analysis

comparing actual project results to planned or expected results. Cost and schedule variance are the most commonly analysed. Others include scope, quality and risk.

Trend Analysis

examining project results over time to determine if performance is improving or deteriorating.

Information Distribution Tools and Techniques

for example, document repositories, meetings, presentations, emails, intranet, databases, etc


Outputs from Performance Reporting

The following are normal outputs from performance reporting:

Performance Reports

which should provide the right information at the right level of detail for various stakeholders as per the Communications Management Plan. They may include bar charts, histograms, tables and any other formats that are best suited to convey project or programme performance information.

Change Requests

analysis of project or programme performance may identify the need for a change to some aspect of the project or programme (for example, scope, schedule, costs). These change requests should be handled as per the Change Control process.
