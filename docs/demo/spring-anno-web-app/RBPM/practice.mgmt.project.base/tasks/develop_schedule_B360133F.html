Task: Develop Schedule

Purpose

To help plan, execute and control project tasks and to track and monitor the progress of the project.

Procedure

  1. Based on the WBS created earlier, determine the tasks that must be included in the schedule, the effort required to complete each task and the dependencies between tasks through:
    • application of experience
    • consultation with the project participants, i.e. the Project Team members and any supporting roles, such as GLT, other shared resources and suppliers
    • consultation with anyone with previous experience of a similar project
    • any other source of useful information that may identify tasks
  2. List each task within the appropriate phase.
  3. Order the tasks that have dependencies which are not related to the availability of the people undertaking the work, e.g. the server must be delivered before it can be configured. This should be achieved by adding dependencies to the tasks within the scheduling tool to simplify keeping the schedule up-to-date as the project progresses.
  4. Assign a resource to each task.
  5. Schedule the tasks according to restraints imposed by all this information.
