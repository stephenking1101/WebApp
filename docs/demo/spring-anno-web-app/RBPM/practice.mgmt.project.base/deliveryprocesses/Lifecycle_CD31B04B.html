Change Project Lifecycle

Purpose

The Change Project Life Cycle is designed to guide Project Managers through project management controls, activities, and tasks and arranging these into a logical and structured sequence. It intends to help Project Managers to efficiently access all the resources they need in order to complete their project correctly and compliantly.

For Transformation please also refer to Transformation Approach.

Note: The Change Project Lifecycle does not affect existing RBPM content - PMI process groups and the standard project controls.

For further information, see Frequently Asked Questions.

Usage

Usage Notes

Each stage in the Project Lifecycle (Definition, Execution, Closure) contains a workflow which depicts the sequential order of activities that a project manager should undertake.

If your project is cancelled / to be cancelled at any point, see Cancelling a Project for additional guidance.
